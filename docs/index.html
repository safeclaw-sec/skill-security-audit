How Sandbox Works

SafeClaw runs the audit in an isolated environment so the skill under test can't access your files, keys, or network. It auto-detects the best available runtime — no manual configuration needed.

1

Docker Linux & macOS Strongest isolation

Full container isolation with no network access, read-only filesystem, 256 MB memory cap, and 0.5 CPU limit. The skill runs in a minimal ephemeral image that is destroyed after the audit.

2

bubblewrap Linux Good isolation

Lightweight userspace sandbox — no root required. Unshares network and PID namespaces, mounts the skill directory read-only, and dies with the parent process. Install with sudo apt install bubblewrap.

3

firejail Linux Good isolation

Established Linux sandboxing tool. Disables network access, uses a private /tmp, and mounts the skill read-only. Slightly heavier than bubblewrap but widely available and well-tested. Install with sudo apt install firejail.

4

sandbox-exec macOS Basic isolation

Native macOS sandbox utility — no installation required. Applies a deny-network policy that blocks all outbound connections during the audit. Lower overhead than Docker; available on every Mac since 10.5.

⚠

Fallback No isolation

When no sandbox is found, SafeClaw presents an interactive install helper that suggests the best option for your OS and shows the exact install commands. In non-interactive mode (CI, cron) it runs with a clear warning. Use --no-sandbox to skip explicitly.

🔍
Self-audit doesn't need a sandbox

When you audit SafeClaw itself, the tool detects it's auditing its own code and runs directly — no sandbox needed. You can read every line of the 1,000+ line source before trusting it.

🛠
No sandbox found? We help you install one

SafeClaw suggests the best option for your OS and shows the exact install commands. You're never left guessing. Runs in CI without prompting — just emits a warning and continues.

🪟
Windows support coming soon

Windows is detected automatically. Until native sandbox support ships, audits run without isolation with a clear warning. WSL2 users can install bubblewrap or Docker and get full isolation today.

⚡
Force a specific runtime

Use --sandbox docker|bwrap|firejail|sandbox-exec to pin a specific runtime, or --no-sandbox to skip isolation entirely for trusted paths.

Quick Start

No API keys. No installation. Just Python 3.8+.

# Clone the repo
git clone https://github.com/safeclaw-sec/skill-security-audit.git

# Audit a skill — auto-picks best sandbox
python3 scripts/sandbox.py /path/to/any/skill

# Force Docker sandbox explicitly
python3 scripts/sandbox.py /path/to/skill --sandbox docker

# Audit without sandbox (trusted paths)
python3 scripts/sandbox.py /path/to/skill --no-sandbox

# Direct audit (no sandbox wrapper)
python3 scripts/audit.py /path/to/skill --no-clawhub

# Self-audit (verify integrity — sandbox auto-skipped)
python3 scripts/sandbox.py .
